# Feedback Notes

ゲームへのフィードバックをここに蓄積します。次回のゲーム作成時に Routine がこのファイルを読んで反映します。

## フォーマット

```
## YYYY-MM-DD ゲーム名 (★1〜5)
- 良かった点:
- 改善点:
- 次に試したいこと:
```

## 共通の好み・方針

（自由に書き込みOK。例: 「文字は大きめが好き」「BGMは無くていい」「縦持ち前提」）

- スマホ縦持ち前提
- 説明なしでも遊べる直感的な操作
- 短時間（30秒〜数分）で1セッション完結
- もう少し複雑でOK。少し個性を出してほしい（例: 見た目の印象を意外な方向にずらす、複数のメカニクスを組み合わせる など）
- 「手触り感」を大事にしたい。タップ/スワイプなど触った瞬間に小さな視覚フィードバック（粒子が散る、ボタンが沈む、波紋が広がる、色がパッと変わる 等）を入れると良い

## 運用ルール（Routineへ）

- **必ず `main` ブランチを基点にして作業すること**（古い作業ブランチを使い回さない）
- 1日1ブランチ・1PRを作成し、**作成後は自身でそのPRを `main` にMergeまで完了させること**
  例: `gh pr create ...` の後に `gh pr merge --merge --delete-branch` を実行
- Mergeできない場合（コンフリクト等）はその旨を `feedback/notes.md` の末尾に簡潔に記録して終了
